使用skill-creater，创建一个产品需求挖掘skill，用于从产品评论中分析迭代需求，具体过程如下：

## Setp 1:关键词分类
**去噪清洗**：删除表情符号、无意义的语气词、闲聊、重复内容，仅保留核心业务信息。
**要素提取**：
   - 用户信息：提取用户的身份、背景或角色（若提及）。
   - 痛点描述：清晰描述用户遇到的具体问题或不便之处。
   - 涉及功能模块：识别该需求归属于产品的哪个功能版块。
**自动分类**：根据内容判定类型，仅限以下四种：[Bug / 建议 / 需求 / 其他]。
**格式输出**：必须以严格的 JSON 格式输出，方便程序解析。
---
{
  "is_valid": true, // 若内容完全无意义则为false
  "cleaned_text": "清洗后的简洁文本",
  "category": "Bug/建议/需求/其他",
  "module": "涉及的功能模块"
}
---

### Setp 2:筛选数据
**选择分类**：使用`AskUserQuestion`工具询问用户需要提取哪一类数据
**格式输出**：将所选分类数据保存为txt文件，一行一条数据

### Setp 3:聚类分析
**特征扫描**：读取txt文件，提取每条数据的“核心意图”、“关键对象”和“独特属性”。
**聚类分组**：
- 相似度阈值：语义相似度 > 85% 或 核心目标一致的项划分为同一簇（Cluster）。
- 识别细微差别：如果两项虽然目标一致但方案不同，需保留并标注为“变体”。
**去重与整合**：
- 保留原则：从每个 Cluster 中选择信息量最全、表达最清晰的一条作为“基准项”。
- 补全逻辑：若其他重复项中含有基准项缺失的独特细节，将其整合进基准项。
**格式输出**：分析结果保存为md文件

### Setp 4: 生成产品优化路线图
**优先级评估**
- 用户价值：解决该问题能惠及多少用户？痛点有多深？
- 实现成本：开发难度是高、中还是低？
- 战略对齐：是否符合当前阶段的产品目标？
**格式输出**：分析结果保存为md文件

## 优先级定义

| 级别 | 定义 | 典型特征 |
|------|------|----------|
| **P0** | 紧急必做 | 影响核心功能、高频使用、用户严重不满、影响业务 |
| **P1** | 重要优先 | 高用户价值、中等成本、符合战略方向 |
| **P2** | 常规优化 | 中等用户价值、常规改进 |
| **P3** | 远期规划 | 低用户价值、高成本、非当前重点 |
